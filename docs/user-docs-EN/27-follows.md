# Follows

A client can follow a provider, an event series, or a saved search to get notifications about what's new.

## Following a provider

By following a provider, you get notifications when they add:

- a **new service**,
- a **new event**,
- a **new promotion**.

## Following an event series

A [recurring event](./17-provider-setup.md) (e.g. weekly walks) can be followed as a **series** — you'll get a notification when the provider **adds a new date**. This is the main way to keep up with the variant that has no dates set in advance.

## Following a search

You can save a search (category, tags, species, location, etc.) and follow it. You'll get a notification when something matching appears:

- **above all, new events** meeting the criteria,
- as well as **new providers / new services** in that area.

## How notifications arrive

Right after the event, but with **spam protection** — if a single provider adds several things at once, you get one combined notification instead of many. The channels (in-app / push / email) depend on your [notification settings](./16-notifications.md).
