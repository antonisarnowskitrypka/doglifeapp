# Contact Support

How any user can get in touch with the platform about a question, problem or bug.

## Submitting a ticket

From within the app, the user sends a request containing:

- **Category** — question / problem / bug
- **Subject** and **description**
- Optional **attachments** (screenshots, files — up to 10 MB each; images and PDF)

## Tracking

- Every ticket has a **status**: open → in progress → resolved.
- The user sees their requests and their current status, and reads the replies.

## Handling

Requests go into the platform operator's queue (see [Super Admin](./24-super-admin.md)), where they are triaged and handled.

## This is not the same as a dispute

Support covers general questions, problems and bugs. A disagreement over whether a **booked service** was performed is a [Dispute](./23-disputes.md), handled within the booking rather than through a support ticket.
