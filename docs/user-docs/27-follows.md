# Obserwowanie

Klient może obserwować usługodawcę, serię wydarzeń albo zapisane wyszukiwanie, żeby dostawać powiadomienia o nowościach.

## Obserwowanie usługodawcy

Obserwując usługodawcę, dostajesz powiadomienia, gdy doda:

- **nową usługę**,
- **nowy event**,
- **nową promocję**.

## Obserwowanie serii wydarzeń

[Wydarzenie powtarzalne](./17-provider-setup.md#wydarzenia-powtarzalne) (np. cotygodniowe spacery) możesz obserwować jako **serię** — dostaniesz powiadomienie, gdy usługodawca **doda nowy termin**. To główny sposób, by nadążyć za wariantem bez ustalonych z góry dat.

## Obserwowanie wyszukiwania

Możesz zapisać wyszukiwanie (kategoria, tagi, gatunek, lokalizacja itd.) i je obserwować. Dostaniesz powiadomienie, gdy pojawi się coś pasującego:

- **przede wszystkim nowe eventy** spełniające kryteria,
- a także **nowi usługodawcy / nowe usługi** w tym obszarze.

## Jak przychodzą powiadomienia

Od razu po zdarzeniu, ale z **ochroną przed spamem** — jeśli jeden usługodawca doda naraz kilka rzeczy, dostaniesz jedno zbiorcze powiadomienie zamiast wielu. Kanały (in-app / push / e-mail) zależą od Twoich [ustawień powiadomień](./16-notifications.md).
